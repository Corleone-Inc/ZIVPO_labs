## Лабораторная работа №4
Работу выполнили студенты Алиев Садык Гаджиибрагимович и Дзюба Даниил Васильевич

### Цель работы: разобрать работу Command&Control -фреймворков

### Ход выполнения работы:

1. Подготовка среды
На виртуальной машине Ubuntu 24.04 LTS была произведена установка фреймворка AdaptixC2. Установка выполнена в соответствии с официальной документацией проекта. ([https://adaptix-framework.gitbook.io/adaptix-framework/]AdaptixC2 Wiki)
Для работы фреймворка были настроены необходимые зависимости.
2.  Настройка Listener и генерация payload
В интерфейсе AdaptixC2 был создан Listener на порту 8080.
После настройки Listener была сгенерирована полезная нагрузка (payload) под целевую операционную систему — в данном случае это была Windows 11.
Payload создан в виде исполняемого файла (server.exe), содержащего встроенный механизм подключения к C2-серверу.
3.  Развертывание агента
Сгенерированный файл server.exe был перемещён на целевую виртуальную машину с ОС Windows 11 в той же локальной сети.
После запуска исполняемого файла на целевой системе произошло установление соединения с сервером AdaptixC2.
В интерфейсе C2-сервера появился новый активный агент, инициированный с IP-адреса целевой машины.
4. Исследование работы фреймворка
После установления соединения были протестированы базовые функции управления агентом:  
- выполнение команд ОС (cp, powershell start, ipconfig, ls, pwd и др.);  
- просмотр информации о системе (ОС, архитектура, пользователь);  
- передача файлов между сервером и агентом.
